# AI 실수 장부

## 목적

이 문서는 Tiny Garden Restore v2부터 사용하는 경량 재발 방지 장부입니다.

Memory Bank가 아니며, 모든 대화나 취향을 누적하는 문서도 아닙니다. 실제 재작업이나 품질 저하로 이어졌고 다음 세션에서도 반복될 가능성이 있는 구체적 실수만 기록합니다.

## 기록 기준

기록 대상은 다음입니다.

- 사용자가 명시적으로 잘못됐다고 지적한 실수
- Unity Play Mode, Console, 테스트, 빌드에서 확인된 실수
- 게임 방향과 다른 기능을 만든 범위 이탈
- 승인되지 않은 설계, 구조, 패키지, 시스템 추가
- 다음 세션에서도 같은 교정을 반복해야 할 가능성이 높은 실수

기록하지 않을 것은 다음입니다.

- 단순 오타
- 한 번만 발생한 사소한 착오
- 취향 수준의 선호
- 아직 검증되지 않은 추측
- 너무 일반적인 조언

## 기록 형식

각 항목은 다음 형식만 사용합니다.

```text
### YYYY-MM-DD - 짧은 제목

- 실수:
- 영향:
- 재발 방지 규칙:
- 관련 근거:
```

## 기록

### 2026-05-21 - v2 범위를 사용자 의도보다 좁게 해석함

- 실수: `docs/project-goal-v2.md` 초안을 작성하면서 v2를 "작은 UX 피드백 + 검증 보강"으로 과도하게 축소했고, 사용자가 원했던 이미지 교체와 AI 실수 장부 생성을 명시적 v2 범위에서 누락했습니다.
- 영향: v2 문서, HTML 시각화, 마일스톤 체크리스트가 사용자 의도와 다르게 UX/검증 중심으로 구성되어 재작성이 필요해졌습니다.
- 재발 방지 규칙: v2처럼 후보 주제가 있는 문서를 작성할 때는 준비 메모의 후보 항목을 `포함 / 제외 / 보류`로 먼저 분류하고, 제외하거나 보류하는 항목은 사용자 의도와 충돌하지 않는지 확인한 뒤 문서화합니다.
- 관련 근거: `docs/reports/project-goal-v2-document-plan-2026-05-20.md`에는 이미지 교체 실험과 `docs/ai-mistakes.md` 검토가 후보로 있었고, 사용자는 v2에 실제 Unity Scene 에셋 교체와 AI 실수 장부 생성을 포함하기로 결정했습니다.
