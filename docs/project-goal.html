codex-harness-tgr-v1

Tiny Garden Restore는 게임 완성보다 하네스 검증이 먼저인 Unity 실험입니다.

목표는 작은 정원 복구 playable loop를 실제로 만들면서, Codex가 얇은 지침과 짧은 프롬프트만으로 계획, 구현, Unity 검증까지 이어갈 수 있는지 확인하는 것입니다.

2D작은 정원 Scene 기반 Unity playable loop
5개1차 검증 대상: AGENTS, Plan mode, MCP, ExecPlan, 짧은 프롬프트
M0-M7문서 정리부터 하네스 평가와 선택 실험까지
Tiny Garden Restore 작게 만들고, 실제로 검증하고, 하네스가 충분한지 판단한다.

완료

0:00 - 1:00

이 프로젝트의 한 문장

읽는 목표게임 자체보다 “최소 하네스가 Unity 작업에 충분한가”를 확인하는 실험이라는 점만 잡으면 됩니다.

무엇을 만드나

방치된 작은 정원을 복구하는 아주 작은 Unity 2D playable loop를 만듭니다.

왜 만드나

복잡한 워크플로우 없이 Codex가 계획, 구현, 검증을 이어갈 수 있는지 확인합니다.

무엇이 아닌가

v0 하네스 확장판이나 상용 수준 게임 구조 검증 프로젝트가 아닙니다.

1:00 - 2:00

게임 범위는 의도적으로 작습니다

핵심 감각작은 정원, 플레이어 이동, 복구 대상 상호작용, 완료 표시만 있으면 초기 목적에 충분합니다.

초기 playable loop에 포함

  • 작은 정원 Scene
  • Player 이동
  • 복구 대상 배치
  • Player와 복구 대상의 상호작용
  • 복구 상태 변화와 완료 표시

초기 단계에서 제외

  • 복잡한 성장 시스템, 경제, 인벤토리
  • 상점, 여러 스테이지, 세이브/로드
  • 복잡한 UI와 고급 아트 파이프라인
  • 상용 수준 구조와 긴 PRD/GDD workflow
  • 여러 agent 협업과 대규모 자동화

2:00 - 3:00

1차 검증 대상은 하네스의 최소 구성입니다

평가 질문적게 갖추고도 실제 Unity 작업이 되는가를 봅니다.

3줄 이하 지시

짧은 작업 프롬프트로도 구현과 검증을 유도할 수 있는지 확인합니다.

3:00 - 3:40

첫 playable loop는 다섯 단계입니다

게임의 최소 끝이동하고, 복구하고, 모든 대상이 끝나면 완료 메시지가 나오는 흐름입니다.

1작은 정원에서 Player가 시작합니다.
2키보드 입력으로 Player가 이동합니다.
3복구 대상 여러 개와 상호작용합니다.
4각 대상의 상태가 복구됨으로 바뀝니다.
5모든 필수 대상 처리 후 완료 메시지를 표시합니다.

3:40 - 4:30

M0부터 M7까지 한 줄로 보기

진행 방향문서 정리에서 시작해 Unity 연결, Scene, 이동, 상호작용, loop 완성, 하네스 평가로 갑니다.

4:30 - 5:00

마지막에 답해야 하는 질문

성공 조건M6까지 끝났을 때 다음 프로젝트에 가져갈 하네스 구성을 판단할 수 있어야 합니다.

성공으로 보는 신호

  • 작은 playable loop가 Unity Play Mode에서 끝까지 동작합니다.
  • Codex가 짧은 프롬프트로 계획, 구현, 검증 결과 보고를 이어갑니다.
  • Unity MCP가 Console log 왕복, package/asset 조회, Scene, Play Mode 확인에 실제로 도움이 됩니다.
  • ExecPlan 없이도 작은 작업을 진행할 수 있는지 판단할 수 있습니다.

실패 또는 보강 신호

  • 작은 작업도 계속 멈추거나 하네스 설정이 작업보다 커집니다.
  • Unity에서 확인 가능한 결과물이 나오지 않습니다.
  • Codex가 요청하지 않은 구조나 기능을 과도하게 추가합니다.
  • 다음 작업으로 이어가기 어려울 정도로 맥락이 사라집니다.

작업 방식

프롬프트는 짧고 검증은 명확하게

운영 원칙한 번에 하나의 목표, 구현 전 plan, 작업 후 변경 파일과 Unity 검증 결과 보고.

좋은 프롬프트 조건

3줄 이하, 목표 하나, 구현 전 계획 요청, 검증 결과 보고 요청, 불필요한 구조화 금지.

기본 형식

[작업 목표]를 해줘.
가장 단순한 방식으로 구현하고, 변경 전 plan을 먼저 제시해줘.
작업 후 변경 파일과 Unity 검증 결과를 보고해줘.

5분 요약: 이 저장소는 “작은 게임을 만들며 최소 Codex 하네스를 검증하는 실험장”입니다.

Tiny Garden Restore의 복잡도를 낮게 유지해야 하네스의 실제 효용이 보입니다. M5까지는 playable loop를 작게 완성하고, M6에서 AGENTS.md, Plan mode, CoderGamester Unity MCP, ExecPlan 참조, 짧은 프롬프트가 각각 유지할 가치가 있는지 판단합니다.
