# Concept 후보

상태: 폐기된 후보 기록

## 목적

이 문서는 브레인스토밍에서 나온 컨셉 후보의 매력, 우려, 선택/보류/폐기 이유를 기록하기 위한 공간이다.

현재 채택한 concept 후보는 없다. 아래 `C-01`은 creative 단계에서 세부 design 사양으로 너무 빨리 수렴한 반례로 폐기한다.

## 후보

### C-01. 작은 분수 복구 노드 퍼즐

상태: 폐기

폐기 이유:

- 사용자가 원한 가벼운 컨셉 탐색보다 세부 gameplay design 사양에 가깝다.
- 행동 횟수, 노드 수, 상태명, UI 조작, 성공/실패 문구가 creative 단계에서 확정된 것처럼 기록되었다.
- 이 후보를 유지하면 이후 design 승인 없이 구현 기준처럼 재사용될 위험이 있다.

아래 내용은 폐기 당시의 후보 기록이다. 현재 채택 후보가 아니며, `docs/design` 승인 없이 gameplay ExecPlan이나 Unity 구현 기준으로 사용할 수 없다.

요약:

```text
작은 노드 맵에서
4회 행동 안에
접근 경로를 열어가며
작은 분수를 복구하는
따뜻한 정원 복구 퍼즐
```

기본 구조:

- 노드 수는 6개로 둔다.
- 플레이어의 행동 횟수는 4회다.
- 각 노드는 `복구하기` 1회 행동으로 복구된다.
- 처음에는 `잡초 더미`만 복구 가능하다.
- `잡초 더미`를 복구하면 `꽃밭`과 `낡은 벤치`가 복구 가능해진다.
- `꽃밭`을 복구하면 `깨진 길`이 복구 가능해진다.
- `깨진 길`을 복구하면 `작은 분수`가 복구 가능해진다.
- `작은 분수`를 복구하면 성공한다.
- `낡은 벤치`는 화면상 정원이 좋아지는 부분 성과 노드지만, 4회 안에 작은 분수를 복구하는 성공 경로에서는 벗어난다.

노드 후보:

```text
입구
잡초 더미
꽃밭
낡은 벤치
깨진 길
작은 분수
```

플레이어에게 보이는 상태:

```text
잠김
복구 가능
복구됨
```

조작 후보:

- 노드를 클릭해서 선택한다.
- 선택한 노드가 `복구 가능`이면 `복구하기` 버튼이 활성화된다.
- `잠김` 또는 `복구됨` 노드는 선택할 수 있지만 `복구하기` 버튼은 비활성화된다.
- 복구하면 행동 횟수가 1 감소한다.
- 새로 열린 노드는 바로 `복구 가능` 상태가 된다.

성공과 실패:

- 성공: `작은 분수가 다시 흐르기 시작했습니다.`
- 실패: `행동이 부족합니다. 다른 복구 순서를 시도해 보세요.`
- Reset 버튼으로 처음 상태로 돌아갈 수 있다.

보류한 확장:

- 자원 종류
- 작업 타입 구분
- 캐릭터 이동
- 명시적 힌트
- 별점, 점수, 보상 화면
- 실제 이미지 에셋 사용
- 세부 연출 구현

## 당시 선택 기준

- 첫 playable에서 4회 안에 성공 또는 실패가 빠르게 판별된다.
- 접근 경로 해금이 눈에 보이는 노드 연결로 설명된다.
- 상태 변화가 `잠김`, `복구 가능`, `복구됨` 3단계로 검증된다.
- 플레이어가 정답을 설명받기보다 연결선과 남은 행동 횟수로 복구 순서를 추론할 수 있다.
- 구현 기준으로 승격하기 전에 `docs/design`에서 별도 승인 절차를 거칠 수 있을 만큼 범위가 작다.

## 당시 보류 또는 폐기 기준

- 첫 playable에서 자원 경제가 필요해지면 보류한다.
- 캐릭터 이동, 충돌, 길찾기 구현이 필요해지면 보류한다.
- 상태 단계나 UI 용어가 플레이어 이해보다 구현 세부를 먼저 드러내면 단순화한다.
- 모든 노드를 복구해야만 성공하는 구조로 바뀌면 첫 playable 범위를 다시 줄인다.
- 실제 이미지 에셋이 필요해지면 `docs/art` 절차로 분리한다.
- 구현 기준으로 쓰려면 `docs/design` 승인 절차로 넘긴다.
