# 브레인스토밍 아이디어 보드

상태: 진행 전

## 목적

이 문서는 Tiny Garden Restore에서 만들 수 있는 여러 게임 아이디어를 카드처럼 쌓는 아이디어 보드다.

각 카드는 확정 기획서가 아니라 후보 메모다. 조작, 규칙, 성공/실패, 화면 아이디어도 자유롭게 적을 수 있지만, `docs/design`으로 승격되기 전에는 gameplay ExecPlan이나 Unity 구현 기준으로 사용할 수 없다.

## 기록 규칙

- 한 게임을 바로 확정하지 않고 여러 아이디어를 나란히 기록한다.
- 카드에는 감정, 소재, 대략의 플레이 상상, 러프한 규칙, 아트/분위기 상상, 좋은 점, 걱정되는 점을 함께 적을 수 있다.
- 초기 보드에는 점수표나 우선순위 평가표를 두지 않는다.
- 상태는 `후보`, `관심 있음`, `보류`, `폐기`, `선택`을 기본으로 쓴다.
- 후보 메모를 구현 기준으로 삼으려면 별도 `docs/design` 절차로 승격해야 한다.

## 카드 형식

```md
### C-00. 아이디어 이름

상태: 후보

- 한 줄 아이디어:
- 플레이어가 느낄 감정:
- 대략의 플레이 상상:
- 아트/분위기 상상:
- 좋은 점:
- 걱정되는 점:
```

## 현재 보드

아직 활성 후보가 없다.

## 폐기된 기록

### C-01. 작은 분수 복구 노드 퍼즐

상태: 폐기

- 한 줄 아이디어: 작은 노드 맵에서 제한된 행동 횟수 안에 접근 경로를 열어가며 작은 분수를 복구하는 퍼즐형 정원 복구.
- 플레이어가 느낄 감정: 버려졌지만 무섭지 않은 작은 정원을 다시 돌보는 조용하고 따뜻한 감정.
- 대략의 플레이 상상: 노드를 고르고 복구하면서 새 경로를 열고, 제한 행동 안에 핵심 오브젝트를 복구한다.
- 아트/분위기 상상: 회색빛으로 방치된 정원에서 시작해 복구할수록 초록색과 밝은 색이 돌아오는 느낌.
- 좋은 점: 상태 변화, 접근 해금, 성공/실패 검증을 작게 볼 수 있었다.
- 걱정되는 점: creative 단계에서 행동 횟수, 노드 수, 상태명, UI 조작, 성공/실패 피드백까지 너무 빨리 확정된 것처럼 기록되었다.

폐기 이유:

- 사용자가 원한 것은 여러 게임 아이디어를 자유롭게 뽑고 고르는 보드였지만, 이 기록은 첫 playable의 세부 gameplay design 사양에 가깝게 수렴했다.
- 이후 세션에서 유효한 후보나 구현 기준으로 오해되지 않도록 폐기된 기록으로만 보존한다.
- 당시 질문과 답변 원문은 `04-brainstorming-qna-2026-05-19.md`에 보존한다.
