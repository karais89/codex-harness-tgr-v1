# 브레인스토밍

상태: 1차 기록

## 목적

이 문서는 Tiny Garden Restore의 아이디어를 넓게 발산하기 위한 공간이다.

## 기록 규칙

- 아이디어는 확정 사양이 아니라 가능성으로 기록한다.
- 선택, 보류, 폐기 판단은 `02-concept-candidates.md`에서 정리한다.
- 플레이어 경험이나 구현 기준으로 확정하려면 별도 design 문서 절차가 필요하다.

## 아이디어

### 2026-05-19: 작은 분수 복구 노드 퍼즐

Tiny Garden Restore의 첫 playable 후보로, 작은 노드 맵에서 제한된 행동 횟수 안에 접근 경로를 열어가며 작은 분수를 복구하는 퍼즐형 정원 복구를 논의했다.

핵심 방향:

- 중심 재미는 복구 순서와 상태 변화가 보이는 작은 퍼즐형 정원 복구다.
- 퍼즐 긴장감은 제한된 행동 횟수 안에서 복구 순서를 고르는 방식에서 만든다.
- 복구 순서는 접근 경로 해금 때문에 중요해진다.
- 공간 표현은 작은 노드 맵 방식으로 둔다.
- 첫 playable의 성공 목표는 제한 행동 안에 핵심 정원 오브젝트 1개를 복구하는 것이다.
- 핵심 오브젝트는 작은 분수다.
- 각 노드는 첫 playable에서 `복구하기` 1회 행동만 사용한다.
- 행동 횟수는 4회다.
- 첫 노드 맵은 6개 노드로 시작하고, 그중 4개가 분수 성공 경로다.
- 낡은 벤치는 실패를 즉시 만들지는 않지만, 분수 성공 경로에서 벗어나는 부분 성과 노드다.
- 플레이어에게 보이는 노드 상태는 `잠김`, `복구 가능`, `복구됨` 3단계다.
- 시각 변화는 첫 playable에서 텍스트, 색상, 간단한 아이콘 변화로만 표현한다.
- 조작은 노드를 클릭해서 선택하고 `복구하기` 버튼을 누르는 방식이다.
- 성공과 실패는 간단한 결과 메시지와 Reset 버튼으로 피드백한다.
- 명시적 힌트는 넣지 않고, 노드 연결선과 잠김 상태만으로 유도한다.
- 노드 이름은 현실적인 정원 오브젝트 이름을 기본으로 한다.
- 분위기는 버려졌지만 무섭지 않은, 조용하고 따뜻한 작은 정원이다.

첫 맵 형태 후보:

```text
[입구]
  |
[잡초 더미]
 /        \
[꽃밭]   [낡은 벤치]
  |
[깨진 길]
  |
[작은 분수]
```

성공 경로 후보:

```text
잡초 더미 -> 꽃밭 -> 깨진 길 -> 작은 분수
```

부분 성과 경로 후보:

```text
잡초 더미 -> 낡은 벤치
```

아직 확정하지 않을 것:

- 자원 종류
- 작업 타입 구분
- 캐릭터 이동
- 별점, 점수, 보상 화면
- 실제 이미지 에셋
- Unity 구현 방식
- player-facing design 승인 여부
