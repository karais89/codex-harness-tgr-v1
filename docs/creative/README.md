# Creative 문서 규칙

## 역할

`docs/creative`는 gameplay design 이전의 아이디어 보드다.

이 폴더에서는 어떤 게임을 만들 수 있을지 여러 아이디어를 카드처럼 꺼내고, 비교하고, 보류하거나 고른다. 한 게임을 너무 빨리 세부 기획서나 첫 playable 사양으로 압축하는 것이 목표가 아니다.

Creative 산출물은 이후 `docs/design` 문서의 재료가 될 수 있지만, 그 자체로 gameplay ExecPlan이나 Unity 구현 기준이 되지 않는다.

## 먼저 읽기

Codex는 다음 작업을 시작하기 전에 이 문서를 읽는다.

- 게임 아이디어 브레인스토밍
- 여러 컨셉 후보 만들기
- 후보의 감정, 소재, 플레이 상상, 우려 기록
- 후보 선택, 보류, 폐기 기록
- concept brief 작성 또는 갱신
- creative 결과를 art 또는 design으로 넘길지 판단하는 작업

## 아이디어 보드 원칙

Creative 단계에서는 아이디어를 넓게 적는다.

- 플레이어에게 남기고 싶은 감정과 인상
- 게임이 떠올리게 하는 장면, 소재, 상황
- 대략의 플레이 상상
- 러프한 조작, 규칙, 성공/실패, 화면 아이디어
- 아트/분위기 상상
- 좋은 점과 걱정되는 점
- 피하고 싶은 톤, 장르 관습, 감각
- 아직 열어 두고 싶은 가능성

조작, 규칙, 성공/실패, 화면 아이디어도 creative 카드에 적을 수 있다. 단, 이 내용은 전부 후보 메모다. `docs/design`으로 승격되기 전에는 확정 사양, 구현 기준, Unity 작업 지시가 아니다.

초기 아이디어 보드에는 점수표나 우선순위 평가표를 두지 않는다. 후보가 충분히 쌓인 뒤 비교 기준이 필요해지면 별도로 논의한다.

## 카드 기본 형식

새 아이디어는 기본적으로 다음 형식을 사용한다.

```md
### C-00. 아이디어 이름

상태: 후보

- 한 줄 아이디어:
- 플레이어가 느낄 감정:
- 대략의 플레이 상상:
- 아트/분위기 상상:
- 좋은 점:
- 걱정되는 점:
```

상태는 `후보`, `관심 있음`, `보류`, `폐기`, `선택` 중 하나를 기본으로 쓴다. 필요하면 짧은 상태를 추가할 수 있지만, 상태만으로 design 승인이나 구현 승인을 뜻하지 않는다.

## 문서 흐름

Creative 작업은 기본적으로 다음 단계별 문서에 기록한다.

1. `01-brainstorming.md`: 여러 게임 아이디어를 카드처럼 쌓는 아이디어 보드다.
2. `02-concept-candidates.md`: 관심 있는 후보의 매력, 우려, 보류/폐기 이유를 조금 더 정리한다.
3. `03-concept-brief.md`: 현재 선택한 컨셉 초안을 짧게 정리한다.

각 문서는 `상태: 작성 전`으로 시작할 수 있다. 이 상태의 문서는 아직 실제 기획 산출물이 아니라 placeholder다.

## Art로 넘어가는 조건

Creative 카드 안에는 `아트/분위기 상상`을 가볍게 적는다.

이미지 프롬프트, 생성 이미지 기록, 시각 레퍼런스 정리는 후보가 선택되었거나 사용자가 명시적으로 더 보고 싶다고 했을 때 `docs/art`로 확장한다.

## Design으로 넘어가는 조건

Creative 문서의 후보 메모를 구현 기준으로 삼으려면 `docs/design` 갱신 절차로 넘어가야 한다.

이때 Codex는 creative 문서를 그대로 확정 사양처럼 사용하지 않는다. 먼저 사용자와 공유된 이해를 검증하고, 명시 승인된 내용만 `docs/design` 문서에 반영한다.

## 엄격 모드

Creative 작업 중 다음이 애매하면 Codex는 파일을 수정하지 않고 사용자에게 먼저 질문한다.

- 새 아이디어 카드를 추가할지, 기존 후보를 선택/보류/폐기할지
- 후보 메모로 남길지, design 기준으로 승격할지
- creative 카드 안의 `아트/분위기 상상`으로 둘지, `docs/art` 작업으로 확장할지
- concept brief를 갱신할지, 후속 design 문서 생성을 논의할지

질문 예:

```text
이 내용은 creative 보드의 후보 메모로 남길까요,
아니면 구현 기준으로 승격하기 위해 docs/design 논의로 넘길까요?
```

## 금지 사항

- Creative 아이디어를 근거로 gameplay 구현을 바로 시작하지 않는다.
- Creative 카드의 후보 메모를 승인된 design 문서처럼 사용하지 않는다.
- `03-concept-brief.md`를 승인된 design 문서처럼 사용하지 않는다.
- 폐기 또는 보류된 아이디어를 조용히 구현 범위에 포함하지 않는다.
- 사용자 승인 없이 creative 산출물을 `상태: 승인됨`인 design 기준으로 바꾸지 않는다.
