v2 기준 문서

v2는 실제 Unity Scene 에셋 교체와 AI 실수 방지를 포함합니다.

v1 playable loop는 유지합니다. v2는 세 복구 대상 이미지를 실제 Scene에 반영하고, 그 뒤 최소 UX 피드백과 Game view 검증을 보강합니다. 동시에 `docs/ai-mistakes.md`를 만들어 Codex 반복 실수를 줄입니다.

AI 실수 장부실수, 영향, 재발 방지 규칙, 관련 근거만 기록합니다.

scope

v2의 세 축

이번 v2는 단순 피드백 보강만이 아니라 실제 이미지 교체와 AI 실수 장부를 명시적 산출물로 포함합니다.

게임 개선

이미지 교체

복구 대상 3개의 새 이미지를 생성/선정하고 Unity Scene에 실제 적용합니다.

실수 방지

AI 실수 장부

반복 가능한 실수를 작게 기록하고 다음 작업의 재발 방지 규칙으로 사용합니다.

검증 보강

UX와 Game view

진행도, E 피드백, 복구 후 시각 피드백을 화면 판독 기준으로 확인합니다.

image replacement

이미지 교체는 문서화에서 끝나지 않습니다

필수 대상은 잡초, 시든 꽃, 더러운 타일입니다. Player는 생성 결과와 작업량을 보고 선택합니다.

교체 흐름

기획 확정 -> 아트 기준 -> 생성 -> 선정 -> Unity 교체

1v1 아트 기준 확인
2교체 목적 정의
3대상별 프롬프트
4이미지 생성
5Unity 사용성 확인
6Scene 교체
제외

이미지 교체와 함께 키우지 않을 시스템

  • 인벤토리, 도구 선택, 대상별 도구 매칭
  • 점수, 콤보, 성장, 경제, 상점
  • 여러 정원, 여러 스테이지, 세이브/로드
  • 복구 순서 퍼즐, 랜덤 배치, 제한 시간
  • memory-bank 또는 큰 workflow 문서

읽는 결론

v2는 Tiny Garden Restore를 크게 키우는 단계가 아닙니다. 하지만 단순 UX 검증만도 아닙니다. 실제 Unity Scene 에셋 교체를 포함한 게임 개선과, `docs/ai-mistakes.md`를 통한 반복 실수 관리가 함께 들어가는 후속 마일스톤입니다.
