# M7 체크리스트 - 선택 실험: Skill / Hooks 검토

## 목적

이 문서는 `docs/project-goal.md`의 `M7. 선택 실험: Skill / Hooks 검토` 완료 기준을 작은 확인 항목으로 나누어, M7 진행 여부와 완료 여부를 체크리스트만으로 판정하기 위해 작성합니다.

M7의 목표는 1차 playable loop 완료 후 M6 평가에서 필요성이 확인된 항목만 작게 실험하는 것입니다. 이 단계는 필수가 아니며, M6 평가 결과에 따라 진행하거나 보류합니다.

## 완료 판정 규칙

- M7은 필수 마일스톤이 아니므로 M6 평가 전에는 `보류`로 둡니다.
- M6 평가에서 Skill 또는 Hooks 필요성이 확인된 항목만 진행합니다.
- 각 실험은 작게 진행해야 하며, playable loop 개선보다 하네스 설정이 커지면 중단합니다.
- 실험 결과가 다음 프로젝트 판단에 도움이 되어야 M7을 완료로 판정합니다.
- M6 평가에서 필요성이 없다고 판단되면, 실험을 하지 않고 `진행하지 않음`으로 판정할 수 있습니다.

## 필수 체크리스트

- [ ] M7-01: M6 평가에서 필요성이 확인된 항목만 진행합니다.
  - 확인 방법: M6 평가 결과에서 Skill / Hooks 필요 여부를 확인합니다.
  - 완료 증거: M6 평가 문서 또는 완료 보고의 판단 내용을 기록합니다.
- [ ] M7-02: Skill 실험 여부를 결정합니다.
  - 확인 방법: 반복되는 Unity 검증 작업이 있었는지 확인합니다.
  - 완료 증거: 진행, 보류, 또는 불필요 판단과 이유를 기록합니다.
- [ ] M7-03: Skill을 진행한다면 작게 만듭니다.
  - 확인 방법: Console error, Player 이동, 복구 상호작용, 완료 메시지 확인 등 반복 검증만 대상으로 삼습니다.
  - 완료 증거: 생성한 skill 경로, 사용한 검증 항목, 실제 사용 결과를 기록합니다.
- [ ] M7-04: Hooks 실험 여부를 결정합니다.
  - 확인 방법: C# 파일 변경 후 반복되는 기계적 검사가 있었는지 확인합니다.
  - 완료 증거: 진행, 보류, 또는 불필요 판단과 이유를 기록합니다.
- [ ] M7-05: Hooks를 진행한다면 보고 전용으로 시작합니다.
  - 확인 방법: 변경 파일, formatting check, lint 또는 analyzer warning, 테스트 명령 안내만 수행하는지 확인합니다.
  - 완료 증거: 생성한 hook 경로, 실행 결과, 자동 수정이 없었다는 확인을 기록합니다.
- [ ] M7-06: 각 실험은 다음 프로젝트 판단에 도움이 됩니다.
  - 확인 방법: 실험 결과가 유지, 폐기, 보류 판단으로 이어지는지 확인합니다.
  - 완료 증거: 다음 프로젝트에서 도입할지에 대한 결론을 기록합니다.
- [ ] M7-07: playable loop 개선보다 하네스 설정이 커지면 중단합니다.
  - 확인 방법: 실험 범위가 게임 구현보다 커지는지 확인합니다.
  - 완료 증거: 중단 여부와 이유, 또는 범위가 작게 유지되었다는 근거를 기록합니다.
