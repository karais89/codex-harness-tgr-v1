# V2-M3 체크리스트 - 이미지 생성과 선정

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M3. 이미지 생성과 선정` 완료 기준을 작은 확인 항목으로 나누어, V2-M2 기준에 따라 실제 이미지 후보를 만들고 Unity Scene에 넣을 후보를 고르기 위해 작성합니다.

## 완료 판정 규칙

- V2-M2 기준 문서가 완료된 뒤에만 진행합니다.
- 필수 복구 대상 3개의 이미지 후보가 있어야 합니다.
- 선정/보류 이유와 Unity 사용 가능성 판단이 기록되어야 합니다.
- 이미지 생성 결과가 없거나 저장 위치가 불명확하면 완료로 판정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M3-01: V2-M2 기준을 확인합니다.
  - 확인 방법: 이미지 교체 목적, 대상별 프롬프트, Unity 사용 가능성 기준을 확인합니다.
  - 완료 증거: 참조한 문서 경로를 기록합니다.
- [ ] V2-M3-02: `잡초` 이미지 후보를 생성합니다.
  - 확인 방법: 생성 결과 파일과 후보 설명을 확인합니다.
  - 완료 증거: 파일 경로, 크기, 선정/보류 판단을 기록합니다.
- [ ] V2-M3-03: `시든 꽃` 이미지 후보를 생성합니다.
  - 확인 방법: 생성 결과 파일과 후보 설명을 확인합니다.
  - 완료 증거: 파일 경로, 크기, 선정/보류 판단을 기록합니다.
- [ ] V2-M3-04: `더러운 타일` 이미지 후보를 생성합니다.
  - 확인 방법: 생성 결과 파일과 후보 설명을 확인합니다.
  - 완료 증거: 파일 경로, 크기, 선정/보류 판단을 기록합니다.
- [ ] V2-M3-05: Player 이미지 후보 생성 여부를 결정합니다.
  - 확인 방법: 선택 대상 Player를 생성할지 보류할지 작업량과 필요성을 기준으로 판단합니다.
  - 완료 증거: 생성 또는 보류 이유를 기록합니다.
- [ ] V2-M3-06: Unity 사용 가능성을 확인합니다.
  - 확인 방법: top-down 판독성, 배경 투명도, Scene 배치 가능성, 파일 크기를 확인합니다.
  - 완료 증거: 선정 후보와 제외 후보를 기록합니다.
- [ ] V2-M3-07: 선정 결과를 문서화합니다.
  - 확인 방법: 어떤 파일을 V2-M4에서 Unity에 넣을지 명확히 기록합니다.
  - 완료 증거: 선정 결과 문서 또는 작업 보고 경로를 기록합니다.
