# V2-M4 체크리스트 - Unity Scene 에셋 교체

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M4. Unity Scene 에셋 교체` 완료 기준을 작은 확인 항목으로 나누어, 선정한 이미지를 실제 Unity Scene에 반영하기 위해 작성합니다.

## 완료 판정 규칙

- V2-M3에서 선정한 이미지 후보만 사용합니다.
- 필수 복구 대상 3개가 실제 Scene에서 새 이미지로 표시되어야 합니다.
- 기존 playable loop를 깨면 완료로 판정하지 않습니다.
- Console error가 있으면 완료로 판정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M4-01: V2-M3 선정 이미지를 확인합니다.
  - 확인 방법: 선정된 파일 경로와 적용 대상을 확인합니다.
  - 완료 증거: 적용할 이미지 목록을 기록합니다.
- [ ] V2-M4-02: Unity 프로젝트에 이미지 에셋을 추가합니다.
  - 확인 방법: 이미지 파일과 `.meta` 파일이 생성되었는지 확인합니다.
  - 완료 증거: 추가된 파일 경로를 기록합니다.
- [ ] V2-M4-03: `잡초` Scene 표현을 교체합니다.
  - 확인 방법: `WeedTargetArea` 또는 대응 오브젝트의 이미지가 교체되었는지 확인합니다.
  - 완료 증거: Scene 오브젝트와 적용 이미지 경로를 기록합니다.
- [ ] V2-M4-04: `시든 꽃` Scene 표현을 교체합니다.
  - 확인 방법: `WiltedFlowerTargetArea` 또는 대응 오브젝트의 이미지가 교체되었는지 확인합니다.
  - 완료 증거: Scene 오브젝트와 적용 이미지 경로를 기록합니다.
- [ ] V2-M4-05: `더러운 타일` Scene 표현을 교체합니다.
  - 확인 방법: `DirtyTileTargetArea` 또는 대응 오브젝트의 이미지가 교체되었는지 확인합니다.
  - 완료 증거: Scene 오브젝트와 적용 이미지 경로를 기록합니다.
- [ ] V2-M4-06: Player 교체 여부를 적용하거나 보류합니다.
  - 확인 방법: V2-M3 판단에 따라 Player 교체 또는 보류를 확인합니다.
  - 완료 증거: 적용/보류 이유와 변경 파일을 기록합니다.
- [ ] V2-M4-07: 기존 playable loop가 유지되는지 확인합니다.
  - 확인 방법: Play Mode에서 시작, 이동, 세 대상 복구, 완료 상태를 확인합니다.
  - 완료 증거: 검증 결과와 Console error 상태를 기록합니다.
