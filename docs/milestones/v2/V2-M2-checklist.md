# V2-M2 체크리스트 - 이미지 교체 기준 확정

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M2. 이미지 교체 기준 확정` 완료 기준을 작은 확인 항목으로 나누어, 실제 이미지 생성 전에 교체 목적과 프롬프트 기준을 고정하기 위해 작성합니다.

## 완료 판정 규칙

- 기존 v1 아트 기준을 확인한 뒤에만 완료로 판정합니다.
- 필수 교체 대상 3개와 선택 대상 Player 기준이 구분되어야 합니다.
- 이미지 생성 프롬프트와 Unity 사용 가능성 판단 기준이 있어야 합니다.
- 이 단계에서는 이미지를 생성하거나 Unity Scene을 수정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M2-01: 기존 v1 아트 기준을 확인합니다.
  - 확인 방법: `docs/art/tiny-garden-art-concept.md`, `docs/art/tiny-garden-image-prompts.md`, `docs/creative/tiny-garden-creative-spec.md`를 확인합니다.
  - 완료 증거: 유지할 아트 기준과 바꿀 필요가 있는 표현을 기록합니다.
- [ ] V2-M2-02: v2 이미지 교체 목적을 정의합니다.
  - 확인 방법: 교체 목적이 "예쁜 이미지"가 아니라 Scene에서 읽히는 복구 대상 표현인지 확인합니다.
  - 완료 증거: 교체 목적 문장을 기록합니다.
- [ ] V2-M2-03: 필수 교체 대상 기준을 정합니다.
  - 확인 방법: `잡초`, `시든 꽃`, `더러운 타일` 각각의 전/후 상태 읽힘 기준을 정합니다.
  - 완료 증거: 대상별 기준을 기록합니다.
- [ ] V2-M2-04: Player 교체 여부 판단 기준을 정합니다.
  - 확인 방법: Player는 선택 대상이며 이미지 생성 결과와 작업량으로 판단한다는 기준을 확인합니다.
  - 완료 증거: Player 교체/보류 기준을 기록합니다.
- [ ] V2-M2-05: 대상별 이미지 생성 프롬프트를 작성합니다.
  - 확인 방법: 필수 대상 3개의 프롬프트가 Unity 2D top-down Scene 사용을 고려하는지 확인합니다.
  - 완료 증거: 프롬프트 문서 경로와 요약을 기록합니다.
- [ ] V2-M2-06: Unity 사용 가능성 판단 기준을 정합니다.
  - 확인 방법: 해상도, 배경 투명도, top-down 판독성, Scene 배치 가능성 기준을 정합니다.
  - 완료 증거: 사용 가능성 체크 기준을 기록합니다.
