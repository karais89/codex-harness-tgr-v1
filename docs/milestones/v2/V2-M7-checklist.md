# V2-M7 체크리스트 - v2 결과 평가

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M7. v2 결과 평가` 완료 기준을 작은 확인 항목으로 나누어, v2 보강이 실제로 v1보다 나아졌는지 판단하기 위해 작성합니다.

## 완료 판정 규칙

- V2-M0부터 V2-M6까지의 결과를 확인한 뒤에만 완료로 판정합니다.
- 평가는 이미지 교체, UX 피드백, 검증 품질, AI 실수 장부, hook/skill 유지 가치에 집중합니다.
- v2에서 하네스가 게임 개선보다 커졌다면 그 위험을 명확히 기록해야 합니다.
- 평가 근거 없이 다음 프로젝트 유지 항목을 확정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M7-01: V2-M0부터 V2-M6까지의 완료 여부를 확인합니다.
  - 확인 방법: 각 V2 체크리스트의 최종 판정과 작업 보고를 확인합니다.
  - 완료 증거: 완료/미완료 항목과 예외를 요약합니다.
- [ ] V2-M7-02: 이미지 교체 결과를 평가합니다.
  - 확인 방법: 교체 이미지가 v1보다 Scene에서 더 잘 읽히는지 확인합니다.
  - 완료 증거: v1 대비 개선, 동일, 악화 중 하나로 기록합니다.
- [ ] V2-M7-03: UX 피드백 결과를 평가합니다.
  - 확인 방법: 진행도, `E: Restore`, 복구 후 피드백이 실제 이해를 돕는지 확인합니다.
  - 완료 증거: 유지할 피드백과 되돌릴 피드백을 기록합니다.
- [ ] V2-M7-04: Game view 또는 screenshot 검증이 효과적이었는지 평가합니다.
  - 확인 방법: V2-M6 검증 결과가 실제 판정에 도움을 줬는지 확인합니다.
  - 완료 증거: 유지할 검증 방식과 버릴 검증 방식을 기록합니다.
- [ ] V2-M7-05: `docs/ai-mistakes.md` 유지 가치를 평가합니다.
  - 확인 방법: 장부가 실제 반복 실수 방지에 도움이 되었는지 확인합니다.
  - 완료 증거: 유지, 수정, 제거, 다음 프로젝트 이관 중 하나로 판단합니다.
- [ ] V2-M7-06: `tiny-garden-playmode-check` skill 유지 가치를 평가합니다.
  - 확인 방법: v2 검증에서 skill이 반복 작업을 줄였는지, 미확인 항목을 명확히 했는지 확인합니다.
  - 완료 증거: 유지, 수정, 제거, 다음 프로젝트 이관 중 하나로 판단합니다.
- [ ] V2-M7-07: Unity MCP preflight hook 유지 가치를 평가합니다.
  - 확인 방법: v2 작업에서 hook이 도움이 되었는지, 혼란이나 비용을 만들었는지 확인합니다.
  - 완료 증거: 유지, 수정, 제거, 문서화 중 하나로 판단하고 이유를 기록합니다.
- [ ] V2-M7-08: 다음 프로젝트로 넘길 판단을 정리합니다.
  - 확인 방법: 유지할 것, 되돌릴 것, 다음 프로젝트에서 다시 검토할 것을 나눕니다.
  - 완료 증거: 최종 평가 보고 또는 v2 종료 보고에 기록합니다.
