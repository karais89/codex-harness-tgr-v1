# V2-M6 체크리스트 - Game view / Play Mode 검증 보강

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M6. Game view / Play Mode 검증 보강` 완료 기준을 작은 확인 항목으로 나누어, v2 결과가 로직으로도 통과하고 화면으로도 읽히는지 확인하기 위해 작성합니다.

## 완료 판정 규칙

- V2-M5까지의 결과를 확인한 뒤에만 완료로 판정합니다.
- Play Mode 로직 검증과 Game view 또는 screenshot 기반 시각 검증을 분리해야 합니다.
- MCP만으로 확인하지 못한 조작감, 화면 판독, 입력 재현 범위는 `미확인`으로 남겨야 합니다.
- 검증하지 못한 항목을 통과처럼 기록하면 완료로 판정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M6-01: V2-M5 결과를 확인합니다.
  - 확인 방법: 이미지 교체와 UX 피드백 보강이 완료되었는지 확인합니다.
  - 완료 증거: 변경 파일과 완료 판정을 기록합니다.
- [ ] V2-M6-02: Unity MCP 또는 동등한 방법으로 Scene 상태를 확인합니다.
  - 확인 방법: active scene path, dirty 상태, 핵심 GameObject 구성을 확인합니다.
  - 완료 증거: Scene path, dirty 상태, 핵심 컴포넌트 확인 결과를 기록합니다.
- [ ] V2-M6-03: Console error를 확인합니다.
  - 확인 방법: Play Mode 검증 전후 Console error를 확인합니다.
  - 완료 증거: Console error 0개 또는 실패 내용을 기록합니다.
- [ ] V2-M6-04: Play Mode에서 전체 loop를 확인합니다.
  - 확인 방법: Player 이동, 세 대상 복구, 완료 상태를 확인합니다.
  - 완료 증거: 실행 방법, 통과 항목, 실패 항목을 기록합니다.
- [ ] V2-M6-05: Game view 또는 screenshot으로 교체 이미지를 확인합니다.
  - 확인 방법: 복구 대상 3개가 화면에서 서로 구분되고 읽히는지 확인합니다.
  - 완료 증거: 캡처 경로 또는 시각 검수 결과를 기록합니다.
- [ ] V2-M6-06: Game view 또는 screenshot으로 UX 피드백을 확인합니다.
  - 확인 방법: 진행도, `E: Restore`, 복구 후 피드백, 완료 표시가 읽히는지 확인합니다.
  - 완료 증거: 캡처 경로 또는 시각 검수 결과를 기록합니다.
- [ ] V2-M6-07: 미확인 항목을 분리합니다.
  - 확인 방법: MCP와 smoke 메뉴만으로 확인하지 못한 조작감, 실제 키 입력, 화면 판독 범위를 분리합니다.
  - 완료 증거: `통과`, `실패`, `미확인`을 나누어 기록합니다.
