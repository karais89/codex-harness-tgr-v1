# V2-M6 체크리스트 - 최소 UX 피드백 보강

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M6. 최소 UX 피드백 보강` 완료 기준을 작은 확인 항목으로 나누어, 복구 상태와 해금 조건 시스템이 화면에서 읽히도록 만들기 위해 작성합니다.

## 완료 판정 규칙

- V2-M5 복구 상태/해금 조건 시스템이 끝난 뒤에만 진행합니다.
- UX 피드백은 새 시스템이 아니라 V2-M5 상태 값을 화면에 드러내는 역할이어야 합니다.
- 인벤토리, 도구 선택, 점수, 성장 같은 확장은 포함하지 않습니다.
- Play Mode에서 진행도, 잠김/복구 가능 상태, 상호작용 피드백을 확인해야 합니다.

## 필수 체크리스트

- [ ] V2-M6-01: V2-M5 결과를 확인합니다.
  - 확인 방법: 복구 가능 상태와 해금 조건이 구현되었는지 확인합니다.
  - 완료 증거: V2-M5 완료 판정 또는 작업 보고를 기록합니다.
- [ ] V2-M6-02: 정원 회복도 표시를 구현합니다.
  - 확인 방법: `0/3`, `1/3`, `2/3`, `3/3` 표시가 복구 상태에 따라 바뀌는지 확인합니다.
  - 완료 증거: 표시 위치, 표시 값 변화, 완료 시 값을 기록합니다.
- [ ] V2-M6-03: `E: Restore` 상호작용 피드백을 구현합니다.
  - 확인 방법: 복구 가능 대상 근처에서만 표시되는지 확인합니다.
  - 완료 증거: 표시 조건과 표시되지 않는 조건을 기록합니다.
- [ ] V2-M6-04: 잠김/복구 가능 상태를 읽히게 합니다.
  - 확인 방법: 아직 해금되지 않은 `시든 꽃`과 복구 가능한 대상이 구분되는지 확인합니다.
  - 완료 증거: 표시 방식과 확인 결과를 기록합니다.
- [ ] V2-M6-05: 복구 후 짧은 시각 피드백을 구현합니다.
  - 확인 방법: 복구 직후 상태 변화가 눈에 읽히는지 확인합니다.
  - 완료 증거: 적용 방식과 확인 결과를 기록합니다.
- [ ] V2-M6-06: 기존 완료 조건을 유지합니다.
  - 확인 방법: 세 대상이 모두 복구되면 완료 상태가 표시되는지 확인합니다.
  - 완료 증거: 완료 조건과 완료 표시 확인 결과를 기록합니다.
- [ ] V2-M6-07: Console error가 없음을 확인합니다.
  - 확인 방법: Play Mode 검증 전후 Console error를 확인합니다.
  - 완료 증거: Console error 0개 결과를 기록합니다.
