# V2-M5 체크리스트 - 복구 상태와 해금 조건 시스템 보강

## 목적

이 문서는 `docs/project-goal-v2.md`의 `V2-M5. 복구 상태와 해금 조건 시스템 보강` 완료 기준을 작은 확인 항목으로 나누어, 기존 복구 루프에 count 기반 해금 조건을 추가하기 위해 작성합니다.

V2-M5의 목표는 v1의 "세 대상에 다가가 E를 누르면 끝나는 흐름"에 한 단계의 이유를 주는 것입니다.

## 완료 판정 규칙

- V2-M4 이미지 교체가 끝난 뒤에만 진행합니다.
- 해금 조건은 단순 count 기반이어야 합니다.
- 도구 선택, 인벤토리, 점수, 시간 제한은 추가하지 않습니다.
- Play Mode에서 복구 가능 상태, 해금 변화, 완료 조건을 확인해야 합니다.
- Console error가 있으면 완료로 판정하지 않습니다.

## 필수 체크리스트

- [ ] V2-M5-01: V2-M4 결과를 확인합니다.
  - 확인 방법: 이미지 교체가 실제 Scene에 반영되었는지 확인합니다.
  - 완료 증거: V2-M4 완료 판정 또는 작업 보고를 기록합니다.
- [ ] V2-M5-02: `RestoreTarget`이 복구 가능 여부를 구분하게 합니다.
  - 확인 방법: 복구 가능 상태와 잠김 상태가 코드에서 구분되는지 확인합니다.
  - 완료 증거: 관련 코드 경로와 상태 판정 방식을 기록합니다.
- [ ] V2-M5-03: `GardenLoopController`가 `restoredCount / totalCount`를 제공하게 합니다.
  - 확인 방법: 현재 복구 수와 전체 대상 수를 UI 또는 다른 시스템에서 사용할 수 있는지 확인합니다.
  - 완료 증거: 관련 코드 경로와 값 확인 결과를 기록합니다.
- [ ] V2-M5-04: 초기 복구 가능 대상을 제한합니다.
  - 확인 방법: `0개 복구됨` 상태에서 `잡초`, `더러운 타일`만 복구 가능한지 확인합니다.
  - 완료 증거: Play Mode 확인 결과를 기록합니다.
- [ ] V2-M5-05: `시든 꽃` 해금 조건을 추가합니다.
  - 확인 방법: `1개 이상 복구됨` 상태에서 `시든 꽃`이 복구 가능해지는지 확인합니다.
  - 완료 증거: Play Mode 확인 결과를 기록합니다.
- [ ] V2-M5-06: 기존 완료 조건을 유지합니다.
  - 확인 방법: 세 대상이 모두 복구되면 `Garden Restored` 완료 상태가 되는지 확인합니다.
  - 완료 증거: 완료 조건 확인 결과를 기록합니다.
- [ ] V2-M5-07: 제외 시스템이 들어가지 않았는지 확인합니다.
  - 확인 방법: 도구 선택, 인벤토리, 점수, 시간 제한, 복잡한 순서 퍼즐이 추가되지 않았는지 확인합니다.
  - 완료 증거: 변경 파일과 제외 확인 결과를 기록합니다.
- [ ] V2-M5-08: Console error가 없음을 확인합니다.
  - 확인 방법: Play Mode 검증 전후 Console error를 확인합니다.
  - 완료 증거: Console error 0개 결과를 기록합니다.
