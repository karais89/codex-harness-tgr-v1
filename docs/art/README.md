# Art 문서 규칙

## 역할

`docs/art`는 아트 방향, 이미지 프롬프트, 생성 이미지 기록을 위한 공간이다.

이 폴더의 문서는 시각 탐색과 판단 근거를 남긴다. 생성 이미지나 프롬프트는 그 자체로 gameplay 사양 또는 Unity 에셋 사용 승인으로 보지 않는다.

Art 작업은 단순 이미지 기록 요청이 아닌 한 `docs/workflows/grill-me.md`의 `Grill-Me Discovery Protocol`로 시작한다. Codex는 이미지를 바로 만들거나 프롬프트를 바로 확정하지 않고, 원하는 시각 방향과 피해야 할 방향을 먼저 질문으로 캐낸다.

## 먼저 읽기

Codex는 다음 작업을 시작하기 전에 이 문서를 읽는다.

- 아트 방향 논의
- 이미지 생성 프롬프트 작성 또는 갱신
- 생성 이미지 저장
- 생성 결과의 채택/보류/폐기 이유 기록
- 이미지 후보를 design 또는 Unity 에셋으로 사용할지 판단하는 작업

이 문서와 함께 `docs/workflows/grill-me.md`를 읽는다.

## 문서 흐름

Art 작업은 기본적으로 다음 단계별 문서에 기록한다.

1. `01-art-direction.md`: 분위기, 색감, 형태 언어, 금지할 시각 방향
2. `02-image-prompts.md`: 실제 이미지 생성 프롬프트와 프롬프트 의도
3. `03-generated-results.md`: 생성 결과, 파일명, 사용한 프롬프트, 채택/보류/폐기 이유
4. `images/`: 채택 또는 검토 가치가 있는 이미지 파일

각 문서는 `상태: 작성 전`으로 시작할 수 있다. 이 상태의 문서는 아직 실제 art 산출물이 아니라 placeholder다.

## Grill-Me Discovery

Codex는 art 작업에서 다음 decision branch가 닫히기 전에는 프롬프트 작성, 이미지 저장, Unity 에셋 사용 판단으로 넘어가지 않는다.

- 원하는 분위기와 피해야 할 분위기
- 색감, 명도, 대비, 형태 언어
- 레퍼런스나 이미지 후보가 art reference인지 player-facing 기준인지
- 이미지 프롬프트가 무엇을 검증하려는지
- 생성 이미지를 저장할지 버릴지
- 저장한 이미지를 채택, 보류, 폐기 중 어디에 둘지
- Unity 에셋 사용 승인 여부

질문은 한 번에 하나만 한다. 각 질문에는 현재 이해, 막힌 결정, 질문, 추천 답안을 포함한다.

## 이미지 저장 기준

`docs/art/images/`에는 채택 또는 검토 가치가 있는 이미지만 저장한다.

실패작, 반복 생성물, 단순 실험 이미지는 기본 저장 대상이 아니다. 저장한 이미지는 `03-generated-results.md`에 다음을 함께 기록한다.

- 파일명
- 사용한 프롬프트 또는 프롬프트 참조
- 채택, 보류, 폐기 상태
- 판단 이유
- Unity 에셋 사용 여부

## 엄격 모드

Art 작업 중 다음이 애매하면 Codex는 파일을 수정하지 않고 사용자에게 먼저 질문한다.

- 단순 art reference인지, player-facing design 기준인지
- 생성 이미지를 저장할지 버릴지
- 이미지가 마음에 든다는 말이 art 후보 채택인지, Unity 에셋 사용 승인인지
- 프롬프트 변경이 art direction만 바꾸는지, 게임의 화면 상태나 피드백 기준까지 바꾸는지

질문 예:

```text
이 이미지는 art reference로만 채택할까요,
아니면 Unity 구현 기준으로도 승인하는 건가요?
```

## Design 또는 Unity 에셋으로 넘어가는 조건

Art 문서의 내용이 플레이어가 보는 규칙, 목표, 조작, 피드백, 화면 상태를 바꾸면 `docs/design` 갱신 절차로 넘어가야 한다.

생성 이미지를 실제 Unity 에셋으로 사용하려면 별도 design 또는 ExecPlan 승인 절차를 거친다. `docs/art/images/`에 파일이 있다는 사실만으로 Unity 에셋 사용이 승인된 것은 아니다.

## 금지 사항

- 생성 이미지를 근거로 gameplay 구현을 바로 시작하지 않는다.
- 프롬프트를 승인된 design 문서처럼 사용하지 않는다.
- 실패작이나 반복 생성물을 무분별하게 커밋하지 않는다.
- 사용자 승인 없이 생성 이미지를 Unity 에셋으로 사용하지 않는다.
