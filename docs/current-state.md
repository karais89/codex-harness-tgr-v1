## 현재 단계

Creative 아이디어 보드 워크플로우 재정의 완료

이 저장소는 Unity 프로젝트와 최소 Codex 하네스를 준비했고, creative/art 워크플로우 문서 구조와 엄격 모드 라우팅 규칙을 적용했습니다. 현재는 기존 creative/art 분류가 사용자가 원한 가벼운 브레인스토밍보다 무거운 게이트처럼 작동한다는 문제를 확인했고, `creative`를 여러 게임 아이디어를 카드처럼 꺼내고 고르는 아이디어 보드로 재정의했습니다. 아직 player-facing gameplay design은 승인되지 않았고, gameplay 구현도 시작하지 않았습니다.

## 활성 계획

`exec-plans/003-creative-idea-board-workflow.md`

이 계획은 creative/art/design 분리 원칙은 유지하되, creative를 승인 게이트가 아니라 여러 게임 아이디어를 자유롭게 만들고 고르는 보드형 브레인스토밍 흐름으로 재정의한 계획입니다. 현재 상태는 `완료`입니다.

## 알려진 제한 사항

- `docs/design/core-beliefs.md`가 `상태: 작성 전`이면 gameplay ExecPlan이나 gameplay 구현 기준으로 사용할 수 없습니다.
- 첫 gameplay 작업은 bootstrap 검증, 승인된 core beliefs, 해당 gameplay에 필요한 승인 design 문서가 생긴 뒤에만 시작할 수 있습니다.
- Creative 산출물은 행동 횟수, 상태명, UI 조작, 성공/실패 조건 같은 gameplay 요소를 러프한 후보 메모로 적을 수 있지만, design 승인 전에는 확정 사양이나 구현 기준이 아닙니다.
